How to Create a GitHub Access Token

To avoid API rate limits on large repositories, you can use a GitHub Personal Access Token (PAT). It's more secure than a password and you can control its permissions.

  1. Go to GitHub Settings:
    Sign in to GitHub. Click your profile picture (top-right), then select Settings.
  2. Navigate to Developer Settings:
    In the left sidebar, scroll down and click on Developer settings.
  3. Select Personal Access Tokens:
    In the new sidebar, click Personal access tokens, then Tokens (classic).
  4. Generate a New Token:
    Click Generate new token and select Generate new token (classic).
  5. Configure Your Token:
    • Note: Give your token a descriptive name (e.g., "RepoVisualizerApp").
    • Expiration: Choose an expiration date (30 days is a good default).
    • Scopes: For this app, you only need to read public data. You can leave all scope boxes unchecked. This is the most secure option.
  6. Create and Copy:
    Scroll down and click the green Generate token button.

Important!

GitHub will only show you the token once. Copy it immediately and paste it into the input box. If you lose it, you must generate a new one.
